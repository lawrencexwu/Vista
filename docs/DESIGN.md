# 選股羅盤（Stock Compass）設計文件

> 目標：把八位可量化投資大師的準則 + 市場溫度計（馬克斯鐘擺 × 科斯托蘭尼雞蛋），
> 做成「主要在 TradingView 上使用」的研究參考工具。每個數字都是研究參考，不是買賣訊號。

---

## 1. 機制選擇：Screener、圖上指標、還是外部批次？

TradingView 上可用的機制有三種，各有硬限制：

| 機制 | 能做什麼 | 硬限制 |
|---|---|---|
| **Pine Screener**（付費方案） | 把自訂 Pine 指標的 `plot()` 當欄位，掃描 watchlist、篩選、排序 | 手動觸發（非每日自動批次）；watchlist 上限 1000 檔；看不到「別檔股票」的數值 → 無法做跨市場百分位排名 |
| **圖上指標（on-chart scorecard）** | 開任何一檔股票，立刻看八個模型各打幾分＋逐條明細 | 一次只看一檔；`request.*` 上限 40 個呼叫 |
| **外部每日批次**（本 repo 的 `batch/` + GitHub Actions） | 全市場掃描、**真正的全市場排名**（神奇公式、RS Rating）、每日固定批次、排行前五、溫度計歷史 | 不在 TradingView 內；結果以報告 + 可匯入 watchlist 的形式回到 TV |

### 逐模組決策矩陣

| 模組 | 純 Pine 可行度 | 決策 | 理由 |
|---|---|---|---|
| 米奈爾維尼 趨勢樣板 | ★★★ | **Pine Screener + 記分板**（原生） | 純價格＋均線＋RS，Pine 主場，8/8 條全部可精確實作 |
| 坦伯頓 極端悲觀點 | ★★★ | **Pine Screener + 記分板** | 「和自己十年歷史比」只需價格與自身估值史 |
| 歐尼爾 CANSLIM | ★★☆ | **Pine 近似版** + 批次補真 RS Rating | C/A/N/M 可實作；L（RS Rating）是跨市場百分位，Pine 用「相對大盤強度」近似，批次算真百分位 |
| 巴菲特 護城河 | ★★☆ | **Pine Screener + 記分板** | ROE/毛利/負債/FCF 欄位 `request.financial` 都有；「長期」受財報歷史長度限制 |
| 林區 PEG | ★★☆ | **Pine Screener + 記分板** | EPS 成長率、PEG 可算；成長分類用門檻 |
| 施洛斯 深度價值 | ★★☆ | **Pine Screener + 記分板** | 低 P/B、低負債、接近多年低點都可 |
| 葛拉漢 防禦型 | ★☆☆ | **Pine 簡化版** + 批次完整版 | 「十年盈餘紀錄、長年股利」在 Pine 受圖表載入歷史限制，簡化為可得年數；批次可用完整財報史 |
| 葛林布萊特 神奇公式 | ★☆☆ | **批次為主**，Pine 只做門檻近似 | 本質是全市場「盈餘殖利率 × 資本報酬率」雙排名，Pine 看不到別檔股票 → 只能顯示兩個原始值＋門檻；真排名在批次 |
| 市場溫度計 | 美股 ★★★ / 台股 ★☆☆ | **Pine 指標（美股用現成 breadth 符號）** + 批次（台股自算 breadth） | 美股有 `INDEX:S5TH`（S&P500 站上 200MA 比例）等符號；台股市場寬度 TV 沒有，批次自己從全市場價格算 |
| 八模型「排行前五」 | ★☆☆ | **批次產出**；TV 內用 Pine Screener 按分數欄排序作即時替代 | 「每日固定批次 + 存檔」只有外部批次做得到 |

### 結論：混合式架構

```
┌────────────────────────────── TradingView ──────────────────────────────┐
│  pine/stock_compass.pine      開任何個股 → 八大師記分板（逐條打勾）        │
│         └─ 同一支指標的 plot() 餵給 Pine Screener → 掃 watchlist、排序     │
│  pine/market_thermometer.pine 開指數圖 → 鐘擺 0–100 + 雞蛋階段            │
└─────────────────────────────────────────────────────────────────────────┘
                ▲  匯入 watchlist（top-5、通過名單）
                │
┌────────────── 本 repo：batch/（GitHub Actions 每日批次）──────────────────┐
│  台股上市＋上櫃全掃、美股 S&P 500 + Nasdaq 100                             │
│  八模型評分 → 各模型排行前五 → reports/YYYY-MM-DD/report_{tw,us}.md        │
│  神奇公式真排名、CANSLIM 真 RS Rating、台股溫度計（自算 breadth）           │
│  產出 TradingView 可匯入的 watchlists/*.txt                                │
└─────────────────────────────────────────────────────────────────────────┘
```

## 2. 評分方式：條件計數制

每個模型定義 N 條準則，逐條打勾，得分 = 通過 X / N 條（並換算成 % 供排序）。
理由：**落差的「原因」一眼可見** —— 台積電在米奈爾維尼 8/8 而巴菲特 3/7 時，
你看得到是哪幾條沒過。這正是「看懂落差」的核心。

資料缺漏（`na`）一律記為**不通過**，但記分板會顯示「資料可得條數」，避免把
缺資料誤讀成壞公司（台股中小型股的基本面欄位在 TV 上常缺）。

## 3. 八模型準則明細（Pine 與批次共用同一套定義）

門檻皆為預設值，Pine 端全部做成 input 可調。

### 3.1 葛拉漢 防禦型投資人（7 條）
| # | 準則 | 資料（TV 財務欄位） |
|---|---|---|
| G1 | 市值夠大（美股 ≥ $2B；台股 ≥ NT$30B，可調） | `close × TOTAL_SHARES_OUTSTANDING` |
| G2 | 流動比率 ≥ 2 | `CURRENT_RATIO` FQ |
| G3 | 長期負債 ≤ 淨營運資金 | `LONG_TERM_DEBT`、`TOTAL_CURRENT_ASSETS − TOTAL_CURRENT_LIABILITIES` FQ |
| G4 | 盈餘穩定：可得年度 EPS 全為正（目標 10 年，依圖表歷史遞減） | `EARNINGS_PER_SHARE_BASIC` FY 序列 |
| G5 | 股利紀錄：可得年度皆配息 | `DPS_COMMON_STOCK_PRIM_ISSUE` FY 序列 |
| G6 | 盈餘成長：最新 FY EPS ≥ 早期 EPS × 1.33（約十年三分之一，依可得年數） | 同 G4 |
| G7 | 價格防線：P/E × P/B ≤ 22.5 | `EARNINGS_PER_SHARE_DILUTED` TTM、`BOOK_VALUE_PER_SHARE` FQ |

### 3.2 巴菲特 質量護城河（7 條）
| # | 準則 | 資料 |
|---|---|---|
| B1 | ROE ≥ 15% | `RETURN_ON_EQUITY` FY |
| B2 | 毛利率 ≥ 40% | `GROSS_MARGIN` TTM |
| B3 | 淨利率 ≥ 10% | `NET_MARGIN` TTM |
| B4 | 負債權益比 ≤ 0.5 | `DEBT_TO_EQUITY` FQ |
| B5 | 自由現金流連續 3 個年度為正 | `FREE_CASH_FLOW` FY 序列 |
| B6 | 營業利益率未走弱（最新 FY ≥ 3 年前 × 0.9） | `OPERATING_MARGIN` FY 序列 |
| B7 | ROA ≥ 7% | `RETURN_ON_ASSETS` FY |

### 3.3 林區 PEG 成長分類（5 條 + 分類標籤）
成長分類（標籤，不計分）：3 年 EPS CAGR < 10% 緩慢；10–20% 穩健；> 20% 快速。
| # | 準則 | 資料 |
|---|---|---|
| L1 | PEG ≤ 1.0（P/E ÷ EPS 3 年 CAGR%） | EPS TTM、EPS FY 序列 |
| L2 | 成長率落在 10%–50%（太低沒故事、太高難持續） | 同上 |
| L3 | 負債權益比 ≤ 0.6 | `DEBT_TO_EQUITY` FQ |
| L4 | P/E 低於自身 5 年中位數 | P/E 序列的 `percentile_nearest_rank` |
| L5 | 營收仍在成長（最新 FY > 前一 FY） | `TOTAL_REVENUE` FY |

### 3.4 葛林布萊特 神奇公式（Pine：2 條門檻近似；批次：真雙排名）
| # | 準則 | 資料 |
|---|---|---|
| M1 | 盈餘殖利率 EBIT/EV ≥ 8% | `OPER_INCOME` TTM ÷ `ENTERPRISE_VALUE` FQ |
| M2 | ROIC ≥ 20%（有形資本報酬近似） | `RETURN_ON_INVESTED_CAPITAL` FY |
批次版：全市場對 EBIT/EV 與 ROIC 各自排名、名次相加 → 綜合排名取前五。

### 3.5 歐尼爾 CANSLIM（7 條，L 與 I 為近似）
| # | 準則 | 資料 |
|---|---|---|
| C1 (C) | 最新季 EPS YoY ≥ 25% | `EARNINGS_PER_SHARE_DILUTED` FQ vs 4 季前 |
| C2 (A) | 年度 EPS 3 年 CAGR ≥ 25% 或 ROE ≥ 17% | FY EPS 序列、ROE |
| C3 (N) | 價格距 52 週高點 ≤ 15% | 價格 |
| C4 (S) | 量能吸籌：10 日均量 > 50 日均量 | 成交量 |
| C5 (L) | 相對強度：6 個月報酬贏過基準指數 ≥ 10 個百分點（Pine 近似；批次算真 RS Rating 百分位 ≥ 70） | 價格 + 指數 |
| C6 (I) | 上漲日量 ≥ 下跌日量（50 日 up/down volume ratio ≥ 1，法人買盤 proxy） | 成交量 |
| C7 (M) | 大盤方向：基準指數 > 其 50MA 且 > 200MA | 指數 |

### 3.6 米奈爾維尼 趨勢樣板（8 條，經典原版）
| # | 準則 |
|---|---|
| T1 | 收盤 > 150MA 且 > 200MA |
| T2 | 150MA > 200MA |
| T3 | 200MA 已上揚至少 1 個月（> 21 個交易日前的自己） |
| T4 | 50MA > 150MA 且 50MA > 200MA |
| T5 | 收盤 > 50MA |
| T6 | 收盤高於 52 週低點至少 30% |
| T7 | 收盤距 52 週高點 ≤ 25% |
| T8 | 相對強度：RS 線（股價/指數）高於 13 週前，且 6 個月贏過指數（RS Rating ≥ 70 的近似） |

### 3.7 施洛斯 深度價值（6 條）
| # | 準則 | 資料 |
|---|---|---|
| S1 | P/B ≤ 1.0 | `BOOK_VALUE_PER_SHARE` FQ |
| S2 | 負債權益比 ≤ 0.3 | `DEBT_TO_EQUITY` FQ |
| S3 | 價格距 3 年低點 ≤ 30% | 價格 |
| S4 | 淨值沒有毀損：股東權益 ≥ 3 年前 | `TOTAL_EQUITY` FY |
| S5 | 有配息 | `DPS_COMMON_STOCK_PRIM_ISSUE` FY |
| S6 | 流動比率 ≥ 1.5（資產底氣） | `CURRENT_RATIO` FQ |

### 3.8 坦伯頓 極端悲觀點（6 條）
| # | 準則 | 資料 |
|---|---|---|
| P1 | 自 10 年（或可得最長）高點回檔 ≥ 40% | 價格 |
| P2 | 位於 10 年價格區間最低 20% | 價格 |
| P3 | P/E ≤ 自身 10 年中位數 × 0.7 | P/E 序列 |
| P4 | 還活著：EPS TTM > 0 或 FCF TTM > 0 | EPS、FCF |
| P5 | 淨值沒有崩壞：BVPS ≥ 5 年前 × 0.8 | `BOOK_VALUE_PER_SHARE` |
| P6 | 已止跌：收盤 ≥ 52 週低點 × 1.05 | 價格 |

## 4. 市場溫度計

**馬克斯鐘擺（0 = 極度悲觀 … 100 = 極度樂觀）** = 以下可得分項的平均，各分項
以 5 年歷史做 `percentrank` 標準化到 0–100：

1. 趨勢熱度：指數距 200MA 的百分比距離的百分位
2. 動能熱度：126 日報酬的百分位
3. 區間位置：52 週區間位置（0–100 直接使用）
4. 市場寬度：站上 200MA 的成分股比例（美股用 `INDEX:S5TH` 等現成符號；台股由批次自算）
5. 波動自滿度：波動率百分位反轉（低波動 = 自滿 = 偏樂觀；美股可改用 VIX）

**科斯托蘭尼雞蛋階段**（六格，由鐘擺分數 × 200MA 趨勢方向分類）：

| 鐘擺 | 趨勢 | 階段 |
|---|---|---|
| ≥ 80 | — | 上升 A3 誇張（過熱） |
| 60–80 | 價 > 200MA | 上升 A2 相隨 |
| 60–80 | 價 < 200MA | 下降 B1 修正（高檔轉弱） |
| 40–60 | 200MA 上揚 | 上升 A1 修正（初升） |
| 40–60 | 200MA 下彎 | 下降 B2 相隨 |
| 20–40 | — | 下降 B2 相隨（加速） |
| < 20 | — | 下降 B3 誇張（過賣） |

這是情境參考，不是買賣訊號。

## 6. 待確認的設計假設（可隨時推翻，架構皆可切換）

互動提問工具在此環境失效，以下用預設值先行，歡迎在對話中直接推翻：

1. **TradingView 方案**：假設 Premium（Pine Screener 全功能、深歷史）。若是免費方案，
   Pine Screener 不可用 → 排行只能靠批次，圖上記分板不受影響。
2. **架構**：採混合式（上表結論）。若想純 TradingView，刪掉 batch/ 即可，代價是
   沒有每日自動批次、真排名與台股溫度計。
3. **市場**：雙市場同時設計；Pine 腳本天生跨市場（同一支腳本開台股美股都能用），
   批次以台股全掃 + 美股兩指數成分為範圍。
4. **評分呈現**：條件計數制（X/N 條 + 逐條打勾），排序用換算百分比。
